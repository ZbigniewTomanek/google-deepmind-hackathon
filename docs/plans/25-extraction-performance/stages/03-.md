# Stage 3: Type Cache+Merge After Ontology

**Goal**: Eliminate the redundant type reload (2 DB calls) after the ontology stage by merging newly created types into the existing lists.
**Dependencies**: Stage 2 (metadata fetch structure)

---

## Steps

1. **Replace type reload with in-memory merge**
   - File: `src/neocortex/extraction/pipeline.py`, lines 116-134
   - Current code:
     ```python
     # 3. Persist new types (skip invalid names)
     for nt in ontology_result.output.new_node_types:
         if await repo.get_or_create_node_type(...) is None:
             logger.warning(...)
     for et in ontology_result.output.new_edge_types:
         if await repo.get_or_create_edge_type(...) is None:
             logger.warning(...)

     # Reload types (now includes newly created)
     node_types = await repo.get_node_types(...)
     edge_types = await repo.get_edge_types(...)
     node_type_descs = {t.name: (t.description or "") for t in node_types}
     edge_type_descs = {t.name: (t.description or "") for t in edge_types}
     ```
   - New approach: capture the return values from `get_or_create_node_type` / `get_or_create_edge_type` and append new types to the existing lists:
     ```python
     # 3. Persist new types and merge into existing lists
     existing_node_names = {t.name for t in node_types}
     for nt in ontology_result.output.new_node_types:
         created = await repo.get_or_create_node_type(
             agent_id, nt.name, nt.description, target_schema=target_schema
         )
         if created is None:
             logger.warning("skipping_invalid_node_type", name=nt.name)
         elif created.name not in existing_node_names:
             node_types.append(created)
             existing_node_names.add(created.name)

     existing_edge_names = {t.name for t in edge_types}
     for et in ontology_result.output.new_edge_types:
         created = await repo.get_or_create_edge_type(
             agent_id, et.name, et.description, target_schema=target_schema
         )
         if created is None:
             logger.warning("skipping_invalid_edge_type", name=et.name)
         elif created.name not in existing_edge_names:
             edge_types.append(created)
             existing_edge_names.add(created.name)

     # Rebuild description dicts with merged types (no reload needed)
     node_type_descs = {t.name: (t.description or "") for t in node_types}
     edge_type_descs = {t.name: (t.description or "") for t in edge_types}
     ```

2. **Return type confirmed**
   - `get_or_create_node_type` returns `NodeType | None` (`db/protocol.py:63-65`), and `get_or_create_edge_type` returns `EdgeType | None`. The append approach works as designed — no special handling needed.
