# Stage 1: Worker Concurrency & Polling Interval

**Goal**: Enable multiple extraction jobs to run in parallel and reduce job pickup latency.
**Dependencies**: None

---

## Steps

1. **Add worker settings to MCPSettings**
   - File: `src/neocortex/mcp_settings.py`
   - Add two new settings after the extraction section (~line 117):
     ```python
     # Worker concurrency & polling
     worker_concurrency: int = 4  # Max parallel extraction jobs
     worker_polling_interval: float = 1.0  # Seconds between job fetch polls
     ```

2. **Pass settings to run_worker_async in MCP server**
   - File: `src/neocortex/server.py`, line 26
   - Change:
     ```python
     job_app.run_worker_async(queues=["extraction"], install_signal_handlers=False)
     ```
   - To:
     ```python
     job_app.run_worker_async(
         queues=["extraction"],
         concurrency=settings.worker_concurrency,
         fetch_job_polling_interval=settings.worker_polling_interval,
         install_signal_handlers=False,
     )
     ```
   - Note: `settings` is already in scope at line 13.

3. **Check if ingestion app also starts a worker**
   - File: `src/neocortex/ingestion/app.py`
   - If a worker is started there too, apply the same concurrency/polling parameters.
   - Per CLAUDE.md: "Start Procrastinate worker in MCP server only (ingestion API just enqueues)" — so likely no changes needed, but verify.
