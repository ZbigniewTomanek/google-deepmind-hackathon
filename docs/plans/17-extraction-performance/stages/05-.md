# Stage 5: Fire-and-Forget Cleanup & Polish

**Goal**: Make `cleanup_empty_types()` non-blocking and add episode-level parallelism within a single extraction job.
**Dependencies**: Stages 2-4

---

## Steps

1. **Fire-and-forget cleanup_empty_types**
   - File: `src/neocortex/extraction/pipeline.py`, lines 194 and 250
   - Currently:
     ```python
     await repo.cleanup_empty_types(agent_id, max_age_minutes=5, target_schema=target_schema)
     ```
   - Change to fire-and-forget via `asyncio.create_task()` (both the tool-driven and fallback paths):
     ```python
     asyncio.create_task(
         repo.cleanup_empty_types(agent_id, max_age_minutes=5, target_schema=target_schema)
     )
     ```
   - Add a try/except wrapper to prevent unhandled exceptions in the background task:
     ```python
     async def _cleanup_bg(repo, agent_id, target_schema):
         try:
             await repo.cleanup_empty_types(agent_id, max_age_minutes=5, target_schema=target_schema)
         except Exception:
             logger.opt(exception=True).warning("cleanup_empty_types_failed")

     asyncio.create_task(_cleanup_bg(repo, agent_id, target_schema))
     ```

2. **Add structured logging for performance observability**
   - File: `src/neocortex/extraction/pipeline.py`
   - Add timing around each pipeline stage to make future optimization measurable:
     ```python
     import time
     t0 = time.monotonic()
     # ... ontology stage ...
     logger.debug("stage_timing", stage="ontology", elapsed_s=round(time.monotonic() - t0, 2))
     ```
   - Instrument: metadata fetch, ontology agent, type persist, extractor agent, embedding pre-compute, librarian agent.
   - Use `logger.debug(...)` so it's visible at DEBUG level but doesn't clutter INFO.

3. **Log worker concurrency at startup**
   - File: `src/neocortex/server.py`
   - After the worker_task creation, log:
     ```python
     logger.info("extraction_worker_started", concurrency=settings.worker_concurrency, poll_interval=settings.worker_polling_interval)
     ```
