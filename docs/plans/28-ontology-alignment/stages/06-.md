**Dependencies**: Stages 1-5 (pipeline must be fixed before cleanup, so re-ingestion produces correct types)

## Steps

### 1. Create cleanup script

- File: `scripts/cleanup_ontology.py` (NEW)
- Details: A standalone asyncio script that connects to the database via asyncpg and
  operates directly with SQL, bypassing the `MemoryRepository` protocol. This is
  intentional — one-shot migration scripts should not depend on application-layer
  abstractions. Follow the pattern established by existing scripts in `scripts/e2e_*.py`.

  The script:

  **a) Removes tool-call artifact types** across all schemas:
  ```sql
  -- Find garbage types
  SELECT id, name FROM {schema}.node_type
  WHERE name ~* '(functiondefault|calldefault|ApicreateOr|UpdateNode|UpdateEdge|createOrUpdate|defaultApi|endcall)';

  -- Reassign nodes to a generic fallback type, then delete the garbage type
  -- Fallback: use 'Concept' for node types, 'RELATES_TO' for edge types
  UPDATE {schema}.node SET type_id = (SELECT id FROM {schema}.node_type WHERE name = 'Concept')
  WHERE type_id IN (SELECT id FROM {schema}.node_type WHERE name ~* '...');
  DELETE FROM {schema}.node_type WHERE name ~* '...';
  ```

  - Log every mutation with before/after counts
  - Run within a transaction per schema (rollback on error)
