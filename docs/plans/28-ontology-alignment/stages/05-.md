# Stage 5: Post-Extraction Type Consolidation

**Goal**: Add a periodic type consolidation mechanism that merges near-duplicate types and archives unused types, preventing drift over time.
**Dependencies**: Stage 1 (validation), Stage 2 (seeds provide canonical type names for merging)

---

## Steps

### 1. Create type consolidation module

- File: `src/neocortex/extraction/type_consolidation.py` (NEW)
- Details: Implement two functions:

  **a) `merge_similar_types(repo, agent_id, schema, dry_run=True)`**

  Finds node types that are likely duplicates and merges them. Merge strategy:
  - Use trigram similarity (already indexed) to find type name pairs with similarity > 0.7
  - Apply a small hardcoded merge map for known equivalences:
    ```python
    _MERGE_MAP = {
        "AnatomicalLocation": "BodyPart",
        "AnatomicalStructure": "BodyPart",
        "HealthActivity": "Activity",
        "SportActivity": "Activity",
        "ArchitectureConcept": "Concept",
        "TechnicalArchitecture": "ArchitecturePattern",
        # ... more from observed data
    }
    ```
  - For each merge: reassign all nodes from source type to target type, then delete source type
  - In dry_run mode, only log what would be merged
  - Return list of (source_type, target_type, node_count) tuples

  **b) `archive_unused_types(repo, agent_id, schema, min_age_hours=24, dry_run=True)`**

  Removes types that have 0 nodes and were created more than `min_age_hours` ago.
  This is more aggressive than the existing `cleanup_empty_types` (which uses 5 min).
  - Query: types with count=0 AND created_at < now() - interval
  - Exclude seed types (from the expanded seed list)
  - In dry_run mode, only log what would be archived
  - Return list of archived type names

### 2. Add consolidation to the extraction pipeline

- File: `src/neocortex/extraction/pipeline.py`
- Details: After every N episodes (configurable, default 10), run
  `archive_unused_types(dry_run=False)` as a fire-and-forget background task.
  This piggybacks on the existing `_cleanup_bg` pattern.

  Do NOT run `merge_similar_types` automatically -- it's destructive and should be
  triggered manually or via the admin API. Add it as an admin endpoint instead.

### 3. Add admin endpoint for type consolidation

- File: `src/neocortex/admin/routes.py`
- Details: Add two endpoints:
  - `POST /admin/consolidate/preview` -- runs both functions in dry_run mode, returns
    what would be merged/archived
  - `POST /admin/consolidate/apply` -- runs both functions for real

  Both accept `schema_name` as a query parameter (or "all" for all schemas).

### 4. Write tests

- File: `tests/test_type_consolidation.py` (NEW)
- Details:
  - Test merge_similar_types with mock repo containing duplicate types
  - Test archive_unused_types with old empty types
  - Test that seed types are never archived
  - Test dry_run mode produces correct output without mutations
