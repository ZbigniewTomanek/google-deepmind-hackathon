# Stage 1: Type Validation Hardening

**Goal**: Add heuristic filters to `normalization.py` that reject tool-call artifacts and instance-level type names before they reach the database.
**Dependencies**: None

---

## Steps

### 1. Add tool-call artifact detection

- File: `src/neocortex/normalization.py`
- Details: Add a blocklist regex that catches patterns like `functiondefault`, `ApicreateOr`,
  `calldefault`, `UpdateNode`, `UpdateEdge`, `content` suffixes when preceded by known function
  fragments. These are Gemini Flash artifacts where internal function call reasoning leaks
  into the output.

  Specific pattern to reject (both node and edge types):
  ```python
  _TOOL_CALL_ARTIFACT = re.compile(
      r"(functiondefault|calldefault|ApicreateOr|UpdateNode|UpdateEdge|"
      r"createOrUpdate|defaultApi|endcall)",
      re.IGNORECASE,
  )
  ```
  Add check in both `normalize_node_type()` and `normalize_edge_type()` — if the pattern
  matches, raise `ValueError("Type name contains tool-call artifact")`.

### 2. Add instance-level type detection for node types

- File: `src/neocortex/normalization.py`
- Details: Detect when a type name looks like a concatenation of a base type + instance name.
  Heuristic: if the name has 3+ PascalCase segments AND the first segment is a known
  common type name (from a small set), reject it.

  Known base types for this heuristic:
  ```python
  _COMMON_BASE_TYPES = {
      "Asset", "Dish", "Dream", "Event", "Insight", "Location",
      "Device", "Activity", "Condition", "Mentalstate", "Utensil",
      "Preparation", "Specification",
  }
  ```

  Logic: split PascalCase name into segments. If `segments[0]` is in `_COMMON_BASE_TYPES`
  and `len(segments) >= 3`, raise `ValueError`. This catches `DishGreg`, `DreamAiPresentation`,
  `LocationSalCapeVerde`, `DeviceMacMiniServer`, `AssetSnowboardguards`,
  `ConditionDurationForFirstFermentation`, etc.

  Exception: Allow 2-segment compound types like `BodyPart`, `HealthState`, `FoodItem` --
  these are legitimate.

### 3. Add minimum length check

- File: `src/neocortex/normalization.py`
- Details: Reject node/edge type names shorter than 2 characters. Single-letter types
  are always garbage.

### 4. Write tests for new validation rules

- File: `tests/test_normalization.py`
- Details: Add test cases covering:
  - Tool-call artifacts are rejected (all observed patterns)
  - Instance-level types are rejected (observed examples)
  - Legitimate compound types pass (`BodyPart`, `HealthState`, `FoodItem`, `MedicalProcedure`)
  - Legitimate new types still pass (`CookingMethod`, `InterpersonalDynamic`, `Supplement`)
  - Edge type artifacts are also rejected
