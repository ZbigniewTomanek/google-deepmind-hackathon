# Stage 4: Model Upgrade for Ontology Agent

**Goal**: Allow the ontology agent to use a stronger model than extraction/curation, since ontology design requires more semantic judgment than Flash with low thinking provides.
**Dependencies**: Stage 3 (prompts must be improved first -- a better model with bad prompts still produces bad types)

---

## Steps

### 1. Add separate ontology model settings

- File: `src/neocortex/mcp_settings.py`
- Details: Add two new settings alongside the existing `ontology_model` and
  `ontology_thinking_effort`:

  ```python
  # Already exists:
  ontology_model: str = "google-gla:gemini-3-flash-preview"
  ontology_thinking_effort: ThinkingLevel = "low"

  # No code change needed IF these already exist and are already used
  # to configure the ontology agent independently.
  ```

  Verify that the existing `ontology_model` and `ontology_thinking_effort` settings
  are actually wired into the ontology agent's config. Check `jobs/tasks.py` where
  `AgentInferenceConfig` is built from settings.

  If they are wired correctly, the only change is the **default value**:
  - Change `ontology_thinking_effort` default from `"low"` to `"medium"`
  - Optionally document that users can set `NEOCORTEX_ONTOLOGY_MODEL` to a stronger
    model like `google-gla:gemini-2.5-pro-preview-06-05` for better ontology quality

### 2. Verify model routing in the pipeline

- File: `src/neocortex/extraction/pipeline.py`
- Details: Confirm that `run_extraction()` uses separate `AgentInferenceConfig` for
  each agent (ontology, extractor, librarian). Currently all three configs are
  passed separately -- verify this is the case and that the ontology config uses
  the dedicated `ontology_model`/`ontology_thinking_effort` settings.

  If the pipeline currently uses the same model for all agents, split it:
  ```python
  ont_cfg = ontology_config or AgentInferenceConfig(
      model_name=settings.ontology_model,
      thinking_effort=settings.ontology_thinking_effort,
  )
  ext_cfg = extractor_config or AgentInferenceConfig(
      model_name=settings.extractor_model,
      thinking_effort=settings.extractor_thinking_effort,
  )
  ```

### 3. Add cost/latency logging for the ontology agent

- File: `src/neocortex/extraction/pipeline.py`
- Details: After the ontology agent run, log the model used and token usage:
  ```python
  logger.info("ontology_agent_complete",
              model=ont_cfg.model_name,
              thinking=ont_cfg.thinking_effort,
              proposed_node_types=len(ontology_result.output.new_node_types),
              proposed_edge_types=len(ontology_result.output.new_edge_types),
              usage=ontology_result.usage())
  ```
  This helps the user compare quality vs. cost when switching models.

### 4. Document model recommendations

- File: `docs/development.md` (or the plan's resources)
- Details: Add a section documenting model recommendations for the ontology agent:
  - Flash + low thinking: fastest, cheapest, worst ontology quality (current default)
  - Flash + medium thinking: moderate cost, significantly better type proposals
  - Gemini Pro / Sonnet: best quality, 3-5x cost, recommended for important data
  - The extractor and librarian should stay on Flash since they work within ontology constraints
