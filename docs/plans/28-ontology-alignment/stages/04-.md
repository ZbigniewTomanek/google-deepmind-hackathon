# Stage 4: Ontology Agent Tuning and Observability

**Goal**: Configure model/thinking settings for the agentic ontology agent, add cost and latency logging, and expose new settings. With tools providing grounding, the model choice is less critical than before — but medium thinking improves tool-use sequencing.
**Dependencies**: Stage 3 (agentic ontology agent must be in place)

---

## Steps

### 1. Upgrade ontology thinking effort default

- File: `src/neocortex/mcp_settings.py`
- Change the default value of `ontology_thinking_effort` from `"low"` to `"medium"`:
  ```python
  ontology_thinking_effort: ThinkingLevel = "medium"  # was "low"
  ```

  **Rationale**: With the agentic design, the ontology agent now plans multi-step
  tool-call sequences (overview → search → propose → retry). Medium thinking helps
  the model sequence these steps correctly. The quality gain from tools is the main
  win, but medium thinking ensures the agent follows its workflow rather than
  short-circuiting to immediate output.

  Note: the model itself (`ontology_model`) stays at Gemini 3 Flash. The agentic
  design reduces the need for a stronger model because tools provide the grounding
  that Flash alone lacked. If quality is still insufficient after the full plan,
  switching to a stronger model is a knob to turn — but try cheaper first.

### 2. Add `ontology_tool_calls_limit` setting

- File: `src/neocortex/mcp_settings.py`
- Add:
  ```python
  ontology_tool_calls_limit: int = 30
  ```

- File: `src/neocortex/jobs/tasks.py` (or wherever `AgentInferenceConfig` is
  constructed for the ontology agent)
- Wire this setting into `run_extraction(ontology_tool_calls_limit=...)`.

### 3. Add cost/latency logging for the ontology agent

- File: `src/neocortex/extraction/pipeline.py`
- After the ontology agent run, log model info, token usage, tool call count,
  and proposal counts:

  ```python
  from pydantic_ai.messages import ToolCallPart

  ontology_tool_calls = sum(
      len(msg.parts)
      for msg in ontology_result.all_messages()
      if hasattr(msg, "parts") and any(isinstance(p, ToolCallPart) for p in msg.parts)
  )
  logger.bind(action_log=True).info(
      "ontology_agent_complete",
      episode_id=episode_id,
      agent_id=agent_id,
      model=ont_cfg.model_name,
      thinking=ont_cfg.thinking_effort,
      proposed_node_types=len(ontology_result.output.new_node_types),
      proposed_edge_types=len(ontology_result.output.new_edge_types),
      tool_calls=ontology_tool_calls,
      model_requests=ontology_result.usage().requests,
      usage=ontology_result.usage(),
  )
  ```

  This enables comparing quality vs. cost across model configurations by querying
  the `agent_actions.log` file.
