# Stage 6: Diagnostic Assessment & Report

**Goal**: Run comprehensive diagnostic queries across all schemas, compile a structured report measuring against Plan 28's success criteria, and identify specific failures that need fixing.
**Dependencies**: Stage 5 DONE

---

## Steps

1. **Ensure all jobs are complete** — no pending extraction work
   ```bash
   curl -s "http://localhost:8001/admin/jobs/summary?all_agents=true" \
     -H "Authorization: Bearer admin-token" | python3 -m json.tool
   ```
   All counts except `done` should be 0. If any `failed` jobs, investigate and document.

2. **Run manual consolidation** on all schemas to ensure cleanup is complete
   ```bash
   # For each schema discovered
   curl -s -X POST "http://localhost:8001/admin/consolidate/apply?schema_name=SCHEMA_NAME_HERE" \
     -H "Authorization: Bearer admin-token" | python3 -m json.tool
   ```

3. **List all schemas** and record their names
   ```bash
   docker compose exec postgres psql -U neocortex -d neocortex -c \
     "SELECT schema_name, agent_id, purpose, is_shared FROM graph_registry ORDER BY schema_name;"
   ```

4. **Run the all-in-one metrics query** for each schema (from `resources/queries.md`)

   Record results in this format per schema:

   ```
   Schema: {schema_name}
   ┌──────────────────────────────┬──────────┬──────────┬────────┐
   │ Metric                       │ Actual   │ Target   │ Status │
   ├──────────────────────────────┼──────────┼──────────┼────────┤
   │ Active node types            │          │ 25-35    │ ?/?    │
   │ Active edge types            │          │ 30-50    │ ?/?    │
   │ Unused edge types (%)        │          │ <15%     │ ?/?    │
   │ Garbage types                │          │ 0        │ ?/?    │
   │ Type reuse ratio             │          │ 20:1+    │ ?/?    │
   └──────────────────────────────┴──────────┴──────────┴────────┘
   ```

   **Note on targets**: The 25-35 node type and 30-50 edge type targets were calibrated
   for ~134 documents. With only 13 test documents, lower counts are expected and acceptable.
   Focus on the ratio-based metrics (unused %, reuse ratio) and zero-tolerance metrics
   (garbage, instance-level).

5. **Run garbage type detection** for all schemas
   ```bash
   docker compose exec postgres psql -U neocortex -d neocortex -c \
     "SELECT name FROM {schema}.node_type
      WHERE name ~* '(functiondefault|calldefault|ApicreateOr|UpdateNode|UpdateEdge|createOrUpdate|defaultApi|endcall)';
      SELECT name FROM {schema}.edge_type
      WHERE name ~* '(functiondefault|calldefault|ApicreateOr|UpdateNode|UpdateEdge|createOrUpdate|defaultApi|endcall)';"
   ```

6. **Run instance-level type detection** for all schemas — list all multi-segment types
   ```bash
   docker compose exec postgres psql -U neocortex -d neocortex -c \
     "SELECT name FROM {schema}.node_type
      WHERE name ~ '^[A-Z][a-z]+[A-Z]'
      ORDER BY name;"
   ```
   Manually review each — determine if it's a legitimate compound type or instance-level.

7. **Check near-duplicate types** across schemas
   ```bash
   docker compose exec postgres psql -U neocortex -d neocortex -c \
     "SELECT a.name, b.name
      FROM {schema}.node_type a
      JOIN {schema}.node_type b ON a.id < b.id
      WHERE LEFT(a.name, 4) = LEFT(b.name, 4)
        AND a.name != b.name
      ORDER BY a.name;"
   ```

8. **Review edge type semantic quality** — list all active edge types across schemas
   ```bash
   docker compose exec postgres psql -U neocortex -d neocortex -c \
     "SELECT et.name, COUNT(e.id) AS usage
      FROM {schema}.edge_type et
      JOIN {schema}.edge e ON e.type_id = et.id
      GROUP BY et.id, et.name
      ORDER BY et.name;"
   ```
   Flag any that seem semantically inappropriate for personal knowledge.

9. **Check ontology agent behavior summary** — aggregate tool usage stats
   ```bash
   grep "ontology_agent_complete" log/agent_actions.log | \
     python3 -c "
   import sys, json
   for line in sys.stdin:
       try:
           d = json.loads(line.split('|')[-1].strip() if '|' in line else line)
           print(f'tools={d.get(\"tool_calls\",\"?\")}, node_types={d.get(\"proposed_node_types\",\"?\")}, edge_types={d.get(\"proposed_edge_types\",\"?\")}')
       except: pass
   "
   ```

10. **Compile the report** — create a summary document with:
    - Per-schema metrics table (from step 4)
    - List of all node types across all schemas with usage counts
    - List of all edge types across all schemas with usage counts
    - List of failures (garbage types, instance-level types, near-duplicates)
    - List of semantically questionable types (manual judgment)
    - Ontology agent behavior summary (avg tool calls, proposal counts)
    - Overall pass/fail assessment per metric

    Save as notes in this plan's index.md Issues section.
