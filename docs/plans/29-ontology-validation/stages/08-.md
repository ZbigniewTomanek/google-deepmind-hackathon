# Stage 8: Re-validation of Failed Scenarios

**Goal**: Start a fresh instance with Stage 7 fixes applied, re-run the specific scenarios that failed in Stage 6, and confirm the fixes resolved the issues.
**Dependencies**: Stage 7 DONE (or SKIPPED if no failures found)

---

## Prerequisites

If Stage 7 was SKIPPED (no failures), this stage can be abbreviated to a
quick confirmation run with just the adversarial documents (Documents 7 and 8)
to double-confirm the system handles edge cases.

---

## Steps

1. **Stop all services**
   ```bash
   ./scripts/manage.sh stop --all
   ```

2. **Start fresh instance** — clean slate with fixes applied
   ```bash
   ./scripts/manage.sh start --fresh
   ```

3. **Clear logs**
   ```bash
   > log/agent_actions.log
   > log/ingestion_stdout.log
   > log/mcp_stdout.log
   ```

4. **Re-ingest only the documents that triggered failures in Stage 6**

   If all documents need re-testing (unlikely), re-run Stages 2-5.
   Otherwise, ingest only the specific documents whose results were problematic.

   At minimum, always re-run the adversarial documents:
   - Document 7 (app names that could leak into types)
   - Document 8 (technical jargon)

   Use the exact same curl commands from Stages 2-4 with `"force": true` added
   to the request body (in case of hash collision with the previous run — though
   fresh instance means new DB, so this shouldn't be needed).

5. **Wait for extraction to complete** — poll `/admin/jobs/summary`

6. **Run the specific diagnostic checks that failed in Stage 6**

   For each failure documented in the Issues section:
   - Run the relevant query from `resources/queries.md`
   - Compare against Stage 6 results
   - Record pass/fail

7. **Run the all-in-one metrics** for final numbers
   ```bash
   # For each schema
   docker compose exec postgres psql -U neocortex -d neocortex -c \
     "<all-in-one query from resources/queries.md>"
   ```

8. **Compile final comparison** — Stage 6 vs Stage 8 side-by-side

   ```
   ┌──────────────────────────────┬──────────┬──────────┬──────────┐
   │ Metric                       │ Stage 6  │ Stage 8  │ Target   │
   ├──────────────────────────────┼──────────┼──────────┼──────────┤
   │ Active node types            │          │          │ 25-35    │
   │ Active edge types            │          │          │ 30-50    │
   │ Unused edge types (%)        │          │          │ <15%     │
   │ Garbage types                │          │          │ 0        │
   │ Instance-level types         │          │          │ 0        │
   │ Type reuse ratio             │          │          │ 20:1+    │
   └──────────────────────────────┴──────────┴──────────┴──────────┘
   ```

9. **Final assessment** — for each metric:
   - **PASS**: Meets target
   - **IMPROVED**: Better than Stage 6 but doesn't meet target yet
   - **UNCHANGED**: Same as Stage 6 (fix didn't help)
   - **REGRESSED**: Worse than Stage 6

10. **Restore original data** (optional, user decision)
    ```bash
    ./scripts/manage.sh stop --all
    ./scripts/manage.sh snapshot load pre-plan29
    ./scripts/manage.sh start
    ```
