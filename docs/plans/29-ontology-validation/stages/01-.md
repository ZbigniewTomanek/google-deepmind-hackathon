# Stage 1: Environment Setup & Data Backup

**Goal**: Backup existing data and start a fresh instance for clean testing.
**Dependencies**: None

---

## Steps

1. **Verify services are running** (or can be started)
   ```bash
   ./scripts/manage.sh status
   ```

2. **Snapshot existing data** — preserves the current database and media store
   so we can restore after testing
   ```bash
   ./scripts/manage.sh snapshot save pre-plan29
   ```
   - If services aren't running, start them first: `./scripts/manage.sh start`
   - Verify snapshot saved: `./scripts/manage.sh snapshot list`

3. **Stop all services**
   ```bash
   ./scripts/manage.sh stop --all
   ```

4. **Start fresh instance** — wipes all data and recreates from scratch
   ```bash
   ./scripts/manage.sh start --fresh
   ```

5. **Verify clean state** — no graphs, no jobs, no data
   ```bash
   # Check no graphs exist yet (should be empty or just seed)
   curl -s http://localhost:8001/admin/graphs \
     -H "Authorization: Bearer admin-token" | python3 -m json.tool

   # Check no jobs exist
   curl -s "http://localhost:8001/admin/jobs/summary?all_agents=true" \
     -H "Authorization: Bearer admin-token" | python3 -m json.tool

   # Check health endpoints
   curl -s http://localhost:8001/health | python3 -m json.tool
   ```

6. **Clear old logs** so we start with clean log files for this test run
   ```bash
   > log/agent_actions.log
   > log/ingestion_stdout.log
   > log/mcp_stdout.log
   ```
