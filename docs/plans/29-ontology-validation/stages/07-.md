# Stage 7: Fix Design & Implementation

**Goal**: Based on the Stage 6 assessment report, design and implement targeted fixes for any failures identified.
**Dependencies**: Stage 6 DONE

---

## Prerequisites

Before starting this stage, review the Issues section of `index.md` populated
during Stage 6. If Stage 6 found zero failures (all metrics pass), mark this
stage as SKIPPED and proceed to Stage 8 for confirmation testing.

---

## Steps

The specific steps depend on what Stage 6 found. Below are fix templates for
each category of failure. Apply only the fixes relevant to identified issues.

### Fix A: Garbage Types Still Leaking

If tool-call artifacts slipped through validation:

1. **Identify the pattern** — what does the garbage type look like?
2. **Update `_TOOL_CALL_ARTIFACT` regex** in `src/neocortex/normalization.py`
   - Add the new pattern to the alternation group
   - Add a test case in `tests/unit/test_normalization.py`
3. **Run unit tests**:
   ```bash
   uv run pytest tests/unit/test_normalization.py -v
   ```

### Fix B: Instance-Level Types Leaking

If types like `ActivityTrackerPro` or entity-name-contaminated types appeared:

1. **Determine if the base type is missing** from `_COMMON_BASE_TYPES` in `normalization.py`
2. **Add missing base types** or **add new entries to whitelist** (`_COMPOUND_TYPE_WHITELIST`)
3. **Consider adding to merge map** in `src/neocortex/extraction/type_consolidation.py`
   if the problem is redundancy rather than garbage
4. **Run unit tests**:
   ```bash
   uv run pytest tests/unit/test_normalization.py -v
   ```

### Fix C: Type Proliferation / Low Reuse

If too many novel types are being created instead of reusing seeds:

1. **Review ontology agent prompt** in `src/neocortex/extraction/agents.py`
   - Strengthen the "reuse over creation" guidance
   - Reduce the type budget if needed (currently max 2 node + 2 edge per episode)
2. **Check if seed types are being surfaced** — the `get_ontology_overview` tool
   should show seed types. If seeds have 0 usage, the agent may not see them as
   viable options.

### Fix D: Semantically Inappropriate Types

If edge types like `BYPASSES_COMPONENT` or `BENCHMARKED_BY` appear:

1. **Add domain-specific guidance** to ontology agent prompt about appropriate
   edge type patterns for personal knowledge
2. **Add problematic patterns to merge map** or archive them manually

### Fix E: Near-Duplicate Types

If types like `Activity` / `HealthActivity` / `SportActivity` coexist:

1. **Add to merge map** in `src/neocortex/extraction/type_consolidation.py`:
   ```python
   _MERGE_MAP: dict[str, str] = {
       # ... existing entries ...
       "NewSource": "ExistingTarget",
   }
   ```
2. **Run unit tests**:
   ```bash
   uv run pytest tests/unit/ -v
   ```

### Fix F: Consolidation Not Triggering

If the archive/merge didn't run as expected:

1. **Check `archive_interval`** in pipeline configuration
2. **Check `_SEED_NODE_TYPES` and `_SEED_EDGE_TYPES`** — seeds are protected
   from archival. Ensure the protection list isn't too broad.
