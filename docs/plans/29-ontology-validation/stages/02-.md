# Stage 2: Single-Document Smoke Test

**Goal**: Verify the basic extraction pipeline works end-to-end by ingesting one document and checking that types are clean.
**Dependencies**: Stage 1 DONE

---

## Steps

1. **Ingest Document 1** (Weekly Note) from `resources/test_documents.md`
   ```bash
   curl -s -X POST http://localhost:8001/ingest/text \
     -H "Content-Type: application/json" \
     -H "Authorization: Bearer claude-code-work" \
     -d '{
       "text": "Weekly Note — March 24-30\n\nHad a great week overall. Monday was productive — finished the kitchen renovation planning with Anna. We decided on quartz countertops from IKEA, estimated cost around 3200 EUR. The contractor Jan starts demolition on April 15.\n\nTuesday I tried a new recipe: shakshuka with feta and za'\''atar. Used the cast iron skillet. Turned out really well — the key is letting the tomato sauce reduce for at least 20 minutes before adding eggs.\n\nWednesday morning run — 7.2km in 38 minutes along the river trail. Felt strong, no knee pain for the first time in weeks. The new Asics Gel-Kayano shoes might be making a difference.\n\nThursday was stressful at work (see work notes). Compensated with a long evening walk and meditation session. Sleep quality was poor — woke up twice, total 6.1 hours.\n\nWeekend trip to Kraków with Marta. Visited Wawel Castle and the Cloth Hall. Had amazing pierogi at a place called Przystanek Pierogarnia near the main square. The obwarzanki from street vendors were surprisingly good too.\n\nMood: generally positive, dip on Thursday. Energy levels: 7/10 average.",
       "metadata": {"source": "plan29-test", "doc": "weekly-note"}
     }' | python3 -m json.tool
   ```
   - Verify response: `status: "stored"`, `episodes_created: 1`

2. **Wait for extraction to complete**
   ```bash
   # Poll until done
   while true; do
     summary=$(curl -s http://localhost:8001/admin/jobs/summary \
       -H "Authorization: Bearer admin-token")
     echo "$summary" | python3 -m json.tool
     todo=$(echo "$summary" | python3 -c "import sys,json; d=json.load(sys.stdin); print(d.get('todo',0)+d.get('doing',0))")
     if [ "$todo" = "0" ]; then echo "Done."; break; fi
     echo "Waiting... ($todo jobs remaining)"; sleep 5
   done
   ```

3. **Check for failed jobs**
   ```bash
   curl -s "http://localhost:8001/admin/jobs?status=failed" \
     -H "Authorization: Bearer admin-token" | python3 -m json.tool
   ```
   - If any jobs failed, check logs: `tail -50 log/ingestion_stdout.log`

4. **Discover created schemas**
   ```bash
   curl -s http://localhost:8001/admin/graphs \
     -H "Authorization: Bearer admin-token" | python3 -m json.tool
   ```
   - Record all schema names for diagnostic queries

5. **Run basic diagnostics on each schema** — see `resources/queries.md`
   ```bash
   # List schemas first
   docker compose exec postgres psql -U neocortex -d neocortex -c \
     "SELECT schema_name, purpose FROM graph_registry ORDER BY schema_name;"

   # For each schema, check node types created
   docker compose exec postgres psql -U neocortex -d neocortex -c \
     "SELECT nt.name, COUNT(n.id) AS usage
      FROM {schema}.node_type nt
      LEFT JOIN {schema}.node n ON n.type_id = nt.id
      GROUP BY nt.id, nt.name
      ORDER BY usage DESC;"
   ```
   Replace `{schema}` with each schema found above.

6. **Check ontology agent logs** — verify tool usage
   ```bash
   grep "ontology_agent_complete" log/agent_actions.log
   ```
   - Look for `tool_calls` > 0 (agent should use tools before proposing)
   - Check `proposed_node_types` and `proposed_edge_types` counts
