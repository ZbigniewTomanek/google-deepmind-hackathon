# Stage 5: Dynamic Seed Generation

**Goal**: Generate or inherit ontology seed recommendations for newly created
domains so extraction has type guidance from the first routed episode.
**Dependencies**: Stage 4 DONE

---

- `src/neocortex/domains/seed_generator.py`

- `src/neocortex/services.py`
- `src/neocortex/jobs/tasks.py`
- `tests/test_seed_generator.py`
- `tests/test_jobs.py`

---

## Steps

1. **Create the seed generator**
   - File: `src/neocortex/domains/seed_generator.py`
   - Add `SeedGenerator(domain_service: DomainService, model: str = "google-gla:gemini-3-flash-preview")`
   - Add `async def resolve_seed(self, slug: str) -> DomainOntologySeed`
   - Resolution order:
     - static seed in `DOMAIN_SEEDS`
     - runtime cache
     - parent inheritance
     - LLM generation

2. **Define parent lookup explicitly**
   - Use `domain_service.get_domain(slug)` for the child
   - Use `domain_service.list_domains()` to build an `id -> domain` map for parent resolution
   - Do not assume a nonexistent `get_domain_by_id()` API

3. **Implement LLM generation**
   - Structured output returning `DomainOntologySeed`
   - Generate 8-15 node types and 8-15 edge types
   - If the domain has a parent, include parent seed types as context

4. **Wire seed resolution into extraction**
   - File: `src/neocortex/extraction/pipeline.py`
   - Add optional parameter:
     - `seed_generator: SeedGenerator | None = None`
   - Replace direct `DOMAIN_SEEDS.get(domain_slug or "")` lookup with:
     - `await seed_generator.resolve_seed(domain_slug)` when available
     - static fallback otherwise

5. **Thread the dependency through services and jobs**
   - File: `src/neocortex/services.py`
   - Add `seed_generator` to `ServiceContext`
   - Construct it after `domain_service`
   - File: `src/neocortex/jobs/tasks.py`
   - Pass `services.get("seed_generator")` into `run_extraction()`
   - Keep the parameter optional so existing direct callers remain valid

6. **Warm the cache after provisioning**
   - File: `src/neocortex/domains/router.py`
   - After a new domain is provisioned, resolve and cache its seed if a generator is available

7. **Add tests**
   - `tests/test_seed_generator.py`
   - `tests/test_jobs.py`
   - Cover:
     - static seed resolution
     - parent inheritance
     - cache hits
     - mocked LLM generation
     - `extract_episode` task passing `seed_generator` through the service context
