# Stage 5: Dynamic Seed Generation

**Goal**: Auto-generate ontology seed recommendations for newly created domains so the extraction pipeline has type guidance from the first episode.
**Dependencies**: Stage 4 DONE

---

## Design

### The Cold-Start Problem

`DOMAIN_SEEDS` in `ontology_seeds.py` only covers 4 hardcoded domains. A newly
provisioned domain starts with zero type recommendations, forcing the ontology
agent to invent types from scratch â€” leading to inconsistent naming.

### Three-Layer Seed Resolution

1. **Static seeds** â€” check `DOMAIN_SEEDS[slug]` (existing behavior)
2. **Parent inheritance** â€” inherit parent domain's seed types as starting point
3. **LLM-generated seeds** â€” cheap structured-output call to generate
   domain-appropriate types from the description + parent context

---

## Steps

1. **Create seed generator module**
   - File: `src/neocortex/domains/seed_generator.py` (new)
   - Constructor: `SeedGenerator(domain_service: DomainService, model: str = "google-gla:gemini-3-flash-preview")`
   - `domain_service` is used to look up domain metadata (name, description, parent)
     so callers only need to pass `slug`
   - Method: `async def resolve_seed(self, slug: str) -> DomainOntologySeed`
     - Looks up domain via `self.domain_service.get_domain(slug)`
     - Resolves parent via `domain.parent_id` if set
   - Resolution order: static â†’ cache â†’ parent inheritance â†’ LLM generation
   - Runtime cache: `dict[str, DomainOntologySeed]`

2. **Implement LLM seed generation**
   - PydanticAI agent with structured output returning `DomainOntologySeed`
   - Prompt: generate 8-15 node types (PascalCase) and 8-15 edge types
     (SCREAMING_SNAKE) appropriate for the domain
   - If parent seed exists, include parent types as context for compatibility
   - Single call per new domain â€” cached thereafter

3. **Wire into extraction pipeline**
   - File: `src/neocortex/extraction/pipeline.py`
   - Replace `seed = DOMAIN_SEEDS.get(domain_slug or "")` (line 140) with
     `seed = await seed_generator.resolve_seed(slug)` if seed_generator is available
   - `seed_generator` is passed as a new optional parameter to `run_extraction()`:
     `seed_generator: SeedGenerator | None = None`
   - Fallback: if `seed_generator` is None, keep existing `DOMAIN_SEEDS.get()` behavior
   - No need to pass domain name/description â€” the generator looks them up internally

4. **Add SeedGenerator to service factory**
   - File: `src/neocortex/services.py`
   - Create after `domain_service` in `create_services()`:
     ```python
     seed_generator = SeedGenerator(domain_service=domain_service)
     ```
   - Add `"seed_generator"` key to `ServiceContext` TypedDict
   - Pass to Procrastinate task handlers via `set_services()`
   - Available to both MCP and ingestion apps

5. **Warm cache on domain provisioning**
   - File: `src/neocortex/domains/router.py`
   - `DomainRouter.__init__()` gains `seed_generator: SeedGenerator | None = None`
   - After `_provision_domain()` creates a domain, call
     `await self.seed_generator.resolve_seed(domain.slug)` to pre-cache the seed
   - First extraction job for the domain will hit cache instead of LLM

6. **Add synchronous fallback in ontology_seeds.py**
   - File: `src/neocortex/domains/ontology_seeds.py`
   - Add `get_seed(slug, parent_slug=None) -> DomainOntologySeed`
   - Does static + parent fallback without LLM (for non-async contexts)

7. **Add tests**
   - File: `tests/test_seed_generator.py` (new)
   - Test static seed resolution
   - Test parent inheritance
   - Test cache hit
   - Test LLM generation with mocked model
   - Validate generated names pass normalization
