# Stage 7: Re-validation

**Goal**: Run the identical 12-document corpus from Stage 2 against the improved system and compare metrics head-to-head to verify elastic upper ontology behavior.
**Dependencies**: Stage 6 DONE

---

## Steps

1. **Fresh start**
   - `./scripts/manage.sh stop --all && ./scripts/manage.sh start --fresh`
   - Confirm 4 seed domains via Q1
   - Confirm hierarchy columns exist: `parent_id`, `depth`, `path`

2. **Ingest all 12 documents**
   - Same commands as Stage 2 — identical corpus, same order
   - Run from `resources/commands.md`
   - Wait for all extraction jobs to complete

3. **Capture post-implementation metrics**

   Run the same diagnostic queries as Stage 2 and record:

   **Q1: Domain registry**
   - Record: total domains, new non-seed domains, their slugs
   - **Key question**: did novel content create new domains?

   **Q6: Domain hierarchy**
   - Record: tree structure, depth levels, parent-child relationships
   - **Key question**: is the hierarchy >1 level deep?

   **Q2/Q3: Episode and node counts per domain**
   - Record: distribution across all domains (including new ones)
   - **Key question**: is content spread more evenly?

   **Q4: Catch-all absorption rate**
   - Record: % of episodes in `domain_knowledge` vs total
   - **Key question**: is catch-all % lower than baseline?

   **Q5: Type landscape for new domains**
   - Record: node types, edge types in newly created domains
   - **Key question**: do new domains have meaningful type vocabularies?

4. **Run taxonomy steward**
   - `./scripts/taxonomy_steward.sh`
   - Record health report
   - Check if any proposals are generated

5. **Head-to-head comparison**

   Fill in the comparison table below with baseline (Stage 2) and post values.

6. **Analyze novel-domain behavior**

   For each Tier 3 document (Docs 07–12):
   - Which domain was it routed to?
   - Was a new domain/sub-domain created?
   - Was the seed ontology useful (check proposed types)?
   - Did extraction produce meaningful nodes/edges in the new domain?

7. **Document any issues found**

   Log issues in the index.md Issues table.
