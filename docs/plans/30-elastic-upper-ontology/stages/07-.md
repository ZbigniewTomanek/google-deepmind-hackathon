**Goal**: Re-run the identical 12-document corpus against the improved system and
compare corrected routing metrics head-to-head.

## Steps

1. **Fresh start**
   - Run `./scripts/manage.sh stop --all && ./scripts/manage.sh start --fresh`
   - Confirm Q1 shows the 4 seed domains plus the hierarchy columns

2. **Ingest all 12 documents**
   - Run the same 12 commands from `resources/commands.md`
   - Wait for `route_episode` and shared-schema `extract_episode` jobs to finish

3. **Capture post-implementation metrics**
   - Q1: domain registry
   - Q2: routed episodes per domain
   - Q3: shared graph utilization per domain
   - Q4: catch-all absorption rate
   - Q5: type landscape for new domains
   - Q6: unmatched / unrouted summary
   - Q7: steward health inputs

4. **Run the taxonomy steward**
   - `./scripts/taxonomy_steward.sh`
   - Capture the report output below

5. **Compare against Stage 2**
   - Use the same corrected metrics
   - Evaluate routing quality, not only domain count

6. **Analyze the 6 novel documents**
   - For Docs 07–12, document:
     - routed domain(s)
     - whether a non-seed domain was created
     - whether the doc still fell into `domain_knowledge`
     - whether it was left unrouted
     - whether the created domain has non-zero graph artifacts

7. **Document issues**
   - Add follow-up issues to the index if any targets are missed
