### Analysis

All 6 novel documents (07–12) triggered the creation of specific non-seed domains.
This is a dramatic improvement over the baseline where 0 non-seed domains were created
and all novel content was absorbed into `domain_knowledge`.

Key improvements:
- **Elastic domain creation works**: 9 non-seed domains were created dynamically,
  6 from the novel docs and 3 from control/cross-domain docs (biology from doc 04,
  health_and_wellness from doc 03, data_science_and_statistics from doc 06).
- **Hierarchy is functional**: 8 of 9 new domains were created as children of
  appropriate seed parents. History was created at root level (no obvious parent).
- **Catch-all pressure reduced**: domain_knowledge absorption dropped from 37.5%
  to 25.8%. Novel docs still route to domain_knowledge as one of multiple targets,
  but they also route to specific new domains.
- **Graph artifacts present**: 12 of 13 domains have non-zero extracted nodes.
  The one exception (data_science_and_statistics) has seed types but 0 extracted
  entities — likely because doc 06 was a cross-domain document whose entities
  mapped better to the other 3 schemas it was routed to.
- **Seed generation quality**: New domains received 24–27 edge types and 2–9 node
  types via the seed generator, meeting the >=5 types target.
- **Steward report is actionable**: The 18 proposals (description_drift +
  underutilized) are reasonable for a 12-doc corpus and provide a clear starting
  point for taxonomy refinement.

One partial miss: the "every created domain has non-zero graph artifacts" target
is not fully met — data_science_and_statistics has seed types but 0 extracted
nodes. This is a minor gap from a cross-domain document, not a systemic failure.
