### Analysis

All 6 novel documents (07–12) were absorbed into `domain_knowledge`. Three of them
(07, 08, 09) were additionally routed to other seed domains, but none triggered the
creation of a new domain. This confirms the root cause: the classifier strongly
prefers existing domains and uses `domain_knowledge` as a catch-all for any content
that doesn't clearly match the other three seeds.

Key observations:
- **Catch-all pressure is high**: `domain_knowledge` absorbed 9 of 24 routed
  episode-domain pairs (37.5%), and ALL 6 novel docs included it.
- **No domain elasticity**: 0 non-seed domains were created despite clearly distinct
  topics (cinema, literature, music, history, cooking, sports).
- **Multi-routing is working**: the classifier does route some docs to multiple
  domains (e.g., Doc 09 went to all 4), but this doesn't help with novel topics.
- **work_context edge extraction gap**: 52 nodes but 0 edges suggests a librarian
  issue specific to that schema's ontology seed guidance.
