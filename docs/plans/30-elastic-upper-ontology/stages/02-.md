# Stage 2: Baseline Experiment

**Goal**: Run the full 12-document corpus on a fresh installation and capture a
correct baseline for current routing behavior.
**Dependencies**: Stage 1 DONE

---

## Files

- `docs/plans/30-elastic-upper-ontology/resources/commands.md`
- `docs/plans/30-elastic-upper-ontology/resources/queries.md`
- `log/agent_actions.log`
- `log/mcp.log`

---

## Steps

1. **Fresh start**
   - Run `./scripts/manage.sh stop --all && ./scripts/manage.sh start --fresh`
   - Confirm the 4 seed domains via Q1

2. **Ingest all 12 documents sequentially**
   - Run all 12 `curl` commands from `resources/commands.md`

3. **Wait for routing/extraction to settle**
   - Poll `procrastinate_jobs` using the commands from `resources/commands.md`
   - Wait until there are no `todo` or `doing` jobs for `route_episode` or shared-schema `extract_episode`
   - Record any failed jobs

4. **Capture baseline metrics**
   - Q1: domain registry and non-seed domains
   - Q2: routed episodes per domain
   - Q3: shared graph utilization per domain
   - Q4: catch-all absorption rate
   - Q5: type landscape
   - Q6: unmatched / unrouted summary from logs

5. **Analyze the 6 novel documents**
   - For Docs 07–12, determine from `log/agent_actions.log` and Q1/Q2:
     - whether a non-seed domain was created
     - whether routing stayed in `domain_knowledge`
     - whether the doc ended up unrouted

6. **Record results in this file**
   - Fill in the results tables below with measured values, not expectations
