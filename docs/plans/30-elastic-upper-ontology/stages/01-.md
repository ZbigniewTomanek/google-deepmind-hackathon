# Stage 1: Test Corpus & Diagnostic Queries

**Goal**: Create the test corpus and diagnostic tooling used in both baseline and re-validation experiments.
**Dependencies**: None

---

## Steps

1. **Review the test corpus** in `resources/test_corpus.md`
   - 12 documents: 4 control (existing domains), 2 cross-domain, 6 novel (cinema, literature, music, history, cooking, sports)
   - Each ~100 words — total corpus ~1200 words, keeping token cost low
   - Already written as part of plan resource creation

2. **Review ingestion commands** in `resources/commands.md`
   - All 12 curl commands ready to run against `localhost:8001`
   - Uses `claude-code-work` dev token
   - Already written as part of plan resource creation

3. **Review diagnostic queries** in `resources/queries.md`
   - Q1–Q7 covering domain registry, episode counts, type landscape, catch-all absorption, hierarchy
   - Baseline capture template included
   - Already written as part of plan resource creation

4. **Verify services can start fresh**
   - Run `./scripts/manage.sh stop --all && ./scripts/manage.sh start --fresh`
   - Confirm all services healthy via `./scripts/manage.sh status`
   - Confirm seed domains exist: run Q1 from queries.md

5. **Dry-run one ingestion** to verify the pipeline works end-to-end
   - Ingest Doc 01 (technical knowledge control)
   - Wait for extraction jobs to complete (watch jobs table)
   - Verify episode appears in `ncx_shared__technical_knowledge.episode`
   - If working, proceed to Stage 2 for the full corpus run
