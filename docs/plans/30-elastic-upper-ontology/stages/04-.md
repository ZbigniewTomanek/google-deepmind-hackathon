# Stage 4: Hierarchical Domain Classifier

**Goal**: Overhaul the domain classifier to display the domain tree, encourage sub-domain proposals, and provide explicit ProposedDomain output guidance.
**Dependencies**: Stage 3 DONE

---

## Design

### Classifier Prompt Principles (OWL-inspired)

1. **Show the taxonomy tree**, not a flat list — the LLM needs structural context
2. **Encourage sub-domain proposals** when content fits broadly under an existing
   domain but deserves specialization
3. **Provide explicit examples** of ProposedDomain with `parent_slug`
4. **Reduce conservatism** — shift from "never propose" to "propose when content
   is clearly outside existing leaf domains"
5. **Kill the catch-all reflex** — `domain_knowledge` should be a last resort,
   not the default for anything unfamiliar

### Tree Display Format

```
Domain tree:
  user_profile: Personal preferences, habits, health, goals
  technical_knowledge: Tools, frameworks, architecture, code
  work_context: Projects, tasks, teams, organizations
  domain_knowledge: General facts, concepts, science
    arts_and_culture: Cinema, literature, music, visual arts
      cinema: Film theory, directors, cinematography
```

## Steps

1. **Add tree formatting utility**
   - File: `src/neocortex/domains/classifier.py`
   - Add function: `format_domain_tree(domains: list[SemanticDomain]) -> str`
   - Build indented tree display using `depth` and `parent_id`
   - Format: `{"  " * depth}{slug}: {description}`

2. **Rewrite AgentDomainClassifier system prompt**
   - File: `src/neocortex/domains/classifier.py`, lines 44-56
   - New prompt (see below). Key changes:
     - Shows domain tree instead of flat list
     - Explicit sub-domain proposal guidance with examples
     - Removes "CONSERVATIVE" directive
     - Adds "DO NOT default to domain_knowledge" rule
     - Provides ProposedDomain JSON structure with parent_slug

   **New system prompt:**
   ```
   You are a knowledge taxonomist for a memory system. Classify incoming
   knowledge into the most specific applicable domain AND propose new
   sub-domains when knowledge genuinely doesn't fit existing categories.

   GUIDELINES:
   - SPECIFIC: route to the deepest matching domain in the tree.
   - MULTI-LABEL: knowledge may belong to multiple domains.
   - PROPOSE SUB-DOMAINS: if knowledge fits broadly under an existing
     domain but deserves specialization, propose a sub-domain.
   - BROAD CATEGORIES: new domains must accumulate many future entries.
     "jazz_piano" is too narrow; "music" is appropriate.
   - DO NOT default to "domain_knowledge" for unfamiliar content.
     Propose a specific sub-domain instead.
   - Confidence >= 0.3 for relevant matches, higher for strong matches.

   DOMAIN TREE:
   {domain_tree}

   PROPOSING NEW DOMAINS:
   Fill proposed_domain with:
   - slug: lowercase_with_underscores, broad category
   - name: Human-readable name
   - description: What belongs here (1-2 sentences)
   - reasoning: Why existing domains don't cover this
   - parent_slug: parent domain slug (null for root-level)

   Good: {slug: "arts_and_culture", parent_slug: null}
   Good: {slug: "cinema", parent_slug: "arts_and_culture"}
   Bad:  {slug: "tarkovsky_films"} (too narrow)
   ```

3. **Update classify() to pass tree**
   - Build tree string via `format_domain_tree(domains)`
   - Inject into prompt's `{domain_tree}` placeholder
   - Increase thinking effort: change `thinking` parameter at `classifier.py:33`
     from `"low"` to `"medium"` in the `AgentDomainClassifier` class attribute

4. **Ensure ProposedDomain has parent_slug**
   - File: `src/neocortex/domains/models.py`
   - Add `parent_slug: str | None = None` if not already present

5. **Update keyword fallback**
   - File: `src/neocortex/domains/classifier.py`
   - Remove `domain_knowledge` as default fallback (line 96) — return empty list instead
   - The keyword fallback must ONLY map to domains that exist in the `domains` list
     passed to `classify()`. Do NOT add keyword groups for domains that don't exist
     yet (like `arts_and_culture`, `history`, `sports`). Novel-domain creation is
     the LLM classifier's job via `proposed_domain`, not the keyword fallback's.
   - The keyword fallback is a safety net for when the LLM fails — it should route
     to existing domains only. When no keywords match, return an empty list so the
     router knows classification produced no result.

6. **Update DomainRouter._provision_domain() for hierarchy**
   - File: `src/neocortex/domains/router.py`
   - If `proposed.parent_slug` is set:
     - Look up parent domain by slug
     - If parent doesn't exist, create parent first as intermediate domain
       (name derived from slug, generic description)
     - Create proposed domain with `parent_id` set
   - Log the full creation chain

7. **Update MockDomainClassifier**
   - For content matching existing domain keywords: return `ClassificationResult`
     with `matched_domains` as before
   - For content matching novel-domain keywords (arts, culture, history, sports,
     cinema, music, cooking): return `ClassificationResult` with `matched_domains=[]`
     and `proposed_domain=ProposedDomain(slug=<inferred>, name=<inferred>,
     description=<inferred>, reasoning="mock", parent_slug=None)`
   - This ensures tests can verify the full provisioning flow deterministically

8. **Update tests**
   - `tests/test_domain_classifier.py`: tree rendering, proposal with parent_slug
   - `tests/test_domain_router.py`: two-level provisioning, parent auto-creation
