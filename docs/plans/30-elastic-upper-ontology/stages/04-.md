# Stage 4: Hierarchical Domain Classifier

**Goal**: Overhaul the classifier to show the domain tree, encourage specific
domain proposals, and stop defaulting unmatched content to `domain_knowledge`.
**Dependencies**: Stage 3 DONE

## Steps

1. **Add a tree-formatting utility**
   - File: `src/neocortex/domains/classifier.py`
   - Add `format_domain_tree(domains: list[SemanticDomain]) -> str`
   - Format each line as `{"  " * depth}{slug}: {description}`

2. **Rewrite the classifier prompt**
   - File: `src/neocortex/domains/classifier.py`
   - Replace the flat-list prompt with a tree-aware prompt
   - Add explicit proposal examples using `parent_slug`
   - Remove the current “CONSERVATIVE” / “prefer existing domains” phrasing
   - Add a rule that `domain_knowledge` is not the default fallback for unfamiliar content

3. **Pass the domain tree into classification**
   - Build the tree string with `format_domain_tree(domains)`
   - Inject it into the prompt

4. **Change the runtime default thinking effort**
   - File: `src/neocortex/mcp_settings.py`
   - Change `domain_classifier_thinking_effort` from `"low"` to `"medium"`
   - Do not rely on changing only the constructor default in `classifier.py`

5. **Update keyword fallback behavior**
   - File: `src/neocortex/domains/classifier.py`
   - Keep keyword fallback only for domains that actually exist in `domains`
   - When neither LLM nor keywords match, return `matched_domains=[]` and `proposed_domain=None`
   - Do not silently force `domain_knowledge`

6. **Update provisioning rules in the router**
   - File: `src/neocortex/domains/router.py`
   - If `proposed.parent_slug` is present, resolve it against an existing domain
   - If the parent slug does not resolve, do **not** auto-create a synthetic parent
   - Either:
     - reject the proposal and log the reason, or
     - treat the proposal as root-level
   - Whichever behavior is chosen, specify it in code comments and tests

7. **Update the mock classifier and tests**
   - `tests/test_domain_classifier.py`
   - `tests/test_domain_router.py`
   - Replace tests that currently assert fallback-to-`domain_knowledge`
   - Add tests for:
     - unmatched text returning no match
     - proposal with `parent_slug`
     - provisioning with existing parent
     - no synthetic parent creation
