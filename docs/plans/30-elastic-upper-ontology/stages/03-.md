# Stage 3: Domain Hierarchy Schema

**Goal**: Add parent-child relationships to `ontology_domains`, enabling hierarchical domain taxonomy with depth tracking and materialized paths.
**Dependencies**: Stage 2 DONE

---

## Design

### OWL Mapping

| OWL Concept | NeoCortex Implementation |
|-------------|--------------------------|
| `rdfs:subClassOf` | `parent_id` foreign key in `ontology_domains` |
| Class hierarchy depth | `depth` integer column (0 = root) |
| Materialized path | `path` TEXT column for efficient subtree queries |

### Hierarchy Rules

1. Root domains have `parent_id = NULL`, `depth = 0`, `path = slug`
2. Sub-domains have `parent_id` pointing to parent, `depth = parent.depth + 1`
3. `path = parent.path || '.' || slug`
4. Maximum depth: 3 levels (configurable, enforced in service layer)
5. Seed domains are always root-level (cannot be moved under a parent)

## Steps

1. **Create migration file**
   - File: `migrations/public/012_domain_hierarchy.sql`

   ```sql
   -- Add hierarchy columns
   ALTER TABLE ontology_domains
       ADD COLUMN IF NOT EXISTS parent_id INTEGER REFERENCES ontology_domains(id),
       ADD COLUMN IF NOT EXISTS depth INTEGER DEFAULT 0,
       ADD COLUMN IF NOT EXISTS path TEXT DEFAULT '';

   -- Backfill existing domains
   UPDATE ontology_domains SET path = slug, depth = 0 WHERE path = '' OR path IS NULL;

   -- Indexes
   CREATE INDEX IF NOT EXISTS idx_ontology_domains_parent ON ontology_domains(parent_id);
   CREATE INDEX IF NOT EXISTS idx_ontology_domains_path ON ontology_domains USING btree(path);
   ```

2. **Update SemanticDomain model**
   - File: `src/neocortex/domains/models.py`
   - Add fields: `parent_id: int | None = None`, `depth: int = 0`, `path: str = ""`
   - Add field: `children: list[SemanticDomain] = []` (populated by tree queries, not stored)

3. **Update ProposedDomain model**
   - File: `src/neocortex/domains/models.py`
   - Add field: `parent_slug: str | None = None` — classifier can propose a sub-domain

4. **Extend DomainService protocol**
   - File: `src/neocortex/domains/protocol.py`
   - Add methods:
     ```python
     async def get_domain_tree(self) -> list[SemanticDomain]:
         """Return root domains with children populated recursively."""

     async def get_children(self, parent_id: int) -> list[SemanticDomain]:
         """Return direct children of a domain."""

     async def move_domain(self, domain_id: int, new_parent_id: int | None) -> SemanticDomain:
         """Move domain under a new parent. Recomputes depth/path for subtree."""
     ```

5. **Implement in PostgresDomainService**
   - File: `src/neocortex/domains/pg_service.py`
   - `create_domain()`: accept `parent_id`, compute `depth` and `path` from parent

   - `get_children()`: `SELECT ... WHERE parent_id = $1 ORDER BY slug`
   - `move_domain()`: update `parent_id`, cascade `depth`/`path` update to subtree
   - `list_domains()`: ensure hierarchy fields are returned

6. **Implement in InMemoryDomainService**
   - File: `src/neocortex/domains/memory_service.py`
   - Mirror PG implementation with in-memory dict

7. **Update seed domain initialization**

8. **Update all callers of `create_domain()`**

     - `tests/test_domain_router.py` — test fixtures creating domains
