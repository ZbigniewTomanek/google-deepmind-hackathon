# Stage 3: Domain Hierarchy Schema

**Goal**: Add parent-child relationships to `ontology_domains`, enabling
hierarchical domain taxonomy for creation-time routing.
**Dependencies**: Stage 2 DONE

Note: this stage does **not** introduce domain reparenting or taxonomy
restructuring. `move_domain()` is intentionally deferred.

## Steps

1. **Create the migration**
   - File: `migrations/public/012_domain_hierarchy.sql`
   - Add columns:
     - `parent_id INTEGER REFERENCES ontology_domains(id)`
     - `depth INTEGER NOT NULL DEFAULT 0`
     - `path TEXT NOT NULL DEFAULT ''`
   - Backfill existing rows with `parent_id = NULL`, `depth = 0`, `path = slug`
   - Add indexes on `parent_id` and `path`

2. **Update the models**
   - File: `src/neocortex/domains/models.py`
   - `SemanticDomain` gains:
     - `parent_id: int | None = None`
     - `depth: int = 0`
     - `path: str = ""`
     - `children: list["SemanticDomain"] = Field(default_factory=list)`
   - `ProposedDomain` gains:
     - `parent_slug: str | None = None`

3. **Extend the protocol**
   - File: `src/neocortex/domains/protocol.py`
   - Add:
     - `get_domain_tree() -> list[SemanticDomain]`
     - `get_children(parent_id: int) -> list[SemanticDomain]`
   - Extend `create_domain()` with `parent_id: int | None = None`

4. **Implement hierarchy in PG and memory services**
   - Files:
     - `src/neocortex/domains/pg_service.py`
     - `src/neocortex/domains/memory_service.py`
   - `create_domain()` computes `depth` and `path` from the parent when provided
   - `list_domains()` and `get_domain()` return hierarchy fields
   - `get_domain_tree()` and `get_children()` are implemented

5. **Update seed defaults and direct callers**
   - Ensure all seed domains initialize as roots with `parent_id=NULL`, `depth=0`, `path=slug`
   - Update direct `create_domain()` call sites and fixtures to pass or accept `parent_id=None`

6. **Update tests**
   - `tests/test_domain_models.py`
   - `tests/test_domain_router.py`
   - `tests/test_domain_e2e.py`
   - Add coverage for sub-domain creation, tree retrieval, and hierarchy defaults

7. **Update the query pack**
   - Update Q1/Q6 references in `resources/queries.md` if needed so hierarchy verification is runnable from the docs
