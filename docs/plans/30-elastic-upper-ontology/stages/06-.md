# Stage 6: Taxonomy Steward CLI

**Goal**: Build a separate process for reviewing domain taxonomy health and proposing restructuring — the maintenance arm of the upper ontology.
**Dependencies**: Stage 5 DONE

---

## Design

### What the Steward Does

The taxonomy steward is a **periodic review process**, not a per-episode classifier.
It runs on-demand (manually via CLI) or on a schedule, and produces a report with
optional actionable proposals. It is the OWL equivalent of an ontology engineer
reviewing the TBox.

### Health Metrics

For each domain, the steward computes:

| Metric | Meaning | Healthy Range |
|--------|---------|---------------|
| **Utilization** | Episodes routed to this domain | >5 episodes |
| **Type diversity** | Number of active node types | 5–30 per domain |
| **Type concentration** | Gini coefficient of type usage | 0.3–0.7 (neither uniform nor dominated by one type) |
| **Catchall ratio** | % of total episodes in domain_knowledge | <30% |
| **Depth utilization** | How many hierarchy levels are populated | ≥2 if >20 episodes |
| **Orphan types** | Types with 0 usage older than 24h | <20% of types |

### Steward Actions (proposals, not auto-applied)

1. **Propose sub-domain split**: when a domain has too many diverse types
   (type diversity > 30 AND low concentration), suggest splitting into sub-domains
2. **Propose domain merge**: when two sibling domains have very similar type
   vocabularies (Jaccard similarity > 0.6 on type names)
3. **Propose domain description update**: when the most-used types suggest the
   domain's focus has drifted from its description
4. **Flag underutilized domains**: domains with <3 episodes after 50+ total
   episodes processed

## Steps

1. **Create steward module**
   - File: `src/neocortex/domains/steward.py` (new)

   ```python
   """Taxonomy steward — domain health review and restructuring proposals."""

   @dataclass
   class DomainHealth:
       slug: str
       name: str
       depth: int
       episode_count: int
       node_count: int
       edge_count: int
       active_node_types: int
       active_edge_types: int
       orphan_node_types: int
       orphan_edge_types: int
       top_types: list[tuple[str, int]]  # (type_name, usage_count)

   @dataclass
   class StewardProposal:
       action: Literal["split", "merge", "update_description", "flag_underutilized"]
       target_domain: str
       reasoning: str
       details: dict  # action-specific data

   class TaxonomySteward:
       async def run_health_check(self) -> list[DomainHealth]:
           """Compute health metrics for all domains."""

       async def propose_restructuring(self, health: list[DomainHealth]) -> list[StewardProposal]:
           """Generate restructuring proposals based on health metrics."""

       def format_report(self, health: list[DomainHealth], proposals: list[StewardProposal]) -> str:
           """Format a human-readable report."""
   ```

2. **Implement health check**
   - Query each domain's schema for:
     - Episode count
     - Node/edge counts
     - Active vs orphan type counts
     - Top-5 types by usage
   - Use the diagnostic queries from `resources/queries.md` as basis
   - Return `DomainHealth` objects

3. **Implement proposal generation**
   - **Split proposal**: if `active_node_types > 30`, use LLM to suggest
     how to partition types into sub-domains (cheap structured call)
   - **Merge proposal**: compare type sets between sibling domains,
     suggest merge if Jaccard > 0.6
   - **Description update**: compare domain description to top-5 type names,
     flag drift if <50% of description keywords appear in types
   - **Underutilization flag**: domain with <3 episodes after system has >50 total

4. **Implement report formatter**
   - Text-based report showing:
     - Domain tree with health indicators
     - Per-domain metrics table
     - Proposals with reasoning
   - Output to stdout and optionally to a file

5. **Create CLI entry point**
   - File: `src/neocortex/domains/steward.py` (add `__main__` block or CLI function)
   - Usage: `uv run python -m neocortex.domains.steward [--apply] [--output report.md]`
   - Default: report only (no changes)
   - `--apply`: execute approved proposals (with confirmation prompts)

6. **Create shell wrapper**
   - File: `scripts/taxonomy_steward.sh` (new)
   - Wrapper that sets up env and runs the steward
   - Usage: `./scripts/taxonomy_steward.sh [--apply]`

7. **Add tests**
   - File: `tests/test_taxonomy_steward.py` (new)
   - Test health computation with mock data
   - Test proposal generation for each action type
   - Test report formatting
   - Test with in-memory domain service (no PG needed)
