# Stage 6: Taxonomy Steward Report

**Goal**: Build a report-only process for reviewing domain taxonomy health after
elastic routing changes land.
**Dependencies**: Stage 5 DONE

Note: this stage is intentionally **report-only**. It does not apply changes and
does not include interactive prompts.

---

## Files

- `src/neocortex/domains/steward.py`
- `scripts/taxonomy_steward.sh`
- `tests/test_taxonomy_steward.py`

---

## Design

The steward is a periodic review tool. It consumes:
- routed-episode counts from `procrastinate_jobs` shared-schema `extract_episode` jobs
- shared-schema nodes/edges/types from each domain schema
- domain hierarchy metadata from `ontology_domains`

It does **not** mutate the taxonomy in this plan.

---

## Steps

1. **Create the steward module**
   - File: `src/neocortex/domains/steward.py`
   - Add:
     - `DomainHealth`
     - `StewardProposal`
     - `TaxonomySteward`

2. **Implement health collection against the real schema**
   - Use routed-episode counts from `procrastinate_jobs` where:
     - `task_name = 'extract_episode'`
     - `status = 'succeeded'`
     - `args->>'target_schema' = domain.schema_name`
   - Use shared-schema graph stats with `schema_scoped_connection(pool, schema_name)`
   - For type usage, join:
     - `node.type_id -> node_type.id`
     - `edge.type_id -> edge_type.id`

3. **Implement report-only proposals**
   - Split proposal:
     - high routed volume + high type diversity + low concentration
   - Merge proposal:
     - sibling domains with similar active type sets
   - Description-drift proposal:
     - top active types are inconsistent with the domain description
   - Underutilization flag:
     - domain receives very little routed traffic relative to peers

4. **Implement report formatting**
   - Include:
     - domain tree
     - routed-episode counts
     - node/edge/type metrics
     - proposals with reasoning

5. **Add a CLI entry point**
   - File: `src/neocortex/domains/steward.py`
   - Usage:
     - `uv run python -m neocortex.domains.steward`
     - `uv run python -m neocortex.domains.steward --output report.md`
   - Do not add `--apply` in this plan

6. **Create a shell wrapper**
   - File: `scripts/taxonomy_steward.sh`
   - Wrapper around the Python module

7. **Add tests**
   - `tests/test_taxonomy_steward.py`
   - Unit-test formatter and proposal heuristics with fixture `DomainHealth` data
   - Manual PG smoke test for health collection against a real database
